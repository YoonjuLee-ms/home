03 · Platform Evolution

데이터 플랫폼은 어떻게 AI 플랫폼으로 진화하는가

리포팅을 넘어 '행동(Action)'으로 — 데이터 플랫폼의 종착점은 AI Agent가 비즈니스를 실행하는 구조입니다.

Before · 리포팅 중심 · 사람이 해석

데이터→ ETL→ Data Warehouse→ Dashboard

After · 행동 중심 · AI가 실행 (Agentic)

데이터→ Lakehouse→ Semantic Layer→ Copilot→ AI Agent→ Business Action

핵심 전환: 'BI 대시보드를 본다' → 'Copilot·AI Agent가 데이터 위에서 의사결정하고 실행한다'.

04 · Competitive Landscape

언제 Snowflake · BigQuery 가 경쟁사가 되는가

경쟁은 '대체'가 아니라 '데이터 중력'에서 시작됩니다 — 공존·연결로 전환해 Microsoft 가치를 더하세요.

Snowflake

경쟁이 되는 순간

· 멀티클라우드 중립 DW 를 표준으로 요구
· SQL 분석·데이터 공유(Marketplace) 중심
· 벤더 중립 거버넌스 선호

BigQuery

경쟁이 되는 순간

· GCP 가 이미 핵심 클라우드
· 서버리스 SQL·실시간 분석 선호
· 데이터 중력이 GCP 에 집중

07 · Pricing Concepts

과금 컨셉 — 이름은 달라도 본질은 하나

네 플랫폼 모두 "연산 능력(컴퓨트)을 시간만큼 빌려 쓰는 값"을 매깁니다. 차이는 딱 두 가지 — ① 쓴 만큼 내나(소비형) vs 용량을 통째로 확보하나(예약형), ② 인프라·스토리지가 요금에 포함되나 vs 따로 붙나.

총비용 = (컴퓨트 단가 × 사용량/확보 용량 × 시간) + 스토리지 + 부대비용(인프라·Egress·Support·운영 인건비)

08 · Interactive Cost Simulator

비용 시뮬레이터 — 입력값만 바꾸면 4개 플랫폼 자동 계산

가동시간·규모·단가를 조정하면 월 비용이 실시간으로 계산됩니다. 원화 환산도 지원합니다.

⚠️ 반드시 알아야 할 함정 — 총액을 그대로 비교하면 안 됩니다

각 플랫폼의 "중간 규모"가 서로 다른 양의 실제 연산을 뜻하기 때문입니다. Snowflake의 Medium(4크레딧/h)은 10-DBU Databricks 클러스터보다 훨씬 큰 연산 할당입니다 — 단위가 정규화돼 있지 않습니다. 올바른 비교는 두 단계입니다:

  1. 같은 작업을 실제로 돌려 각 플랫폼이 몇 시간·몇 DBU/크레딧/슬롯을 쓰는지 측정(벤치마크)
  2. 그 실측치를 계산식에 넣어 TCO 산출 — 큰 웨어하우스가 절반 시간에 끝나면 쿼리당 비용은 오히려 더 쌀 수 있습니다.

→ TCO 계산 시 컴퓨트 외 Storage · Egress · Support · 운영 인건비를 반드시 포함하세요.
